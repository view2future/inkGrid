# 项目工作流规范

## 1. 核心流程
本项目遵循**任务驱动开发**流程：
1. **定义任务 (Task)**：在 `plan.md` 中明确任务目标。
2. **测试先行 (TDD)**：针对核心算法（如 OCR 定位、SAM 分割边缘）编写测试用例，要求覆盖率 > 80%。
3. **实现 (Implementation)**：编写业务代码。
4. **验证与提交 (Verify & Commit)**：通过测试后立即提交代码。

## 2. Git 提交规范
- 每次完成 `plan.md` 中的一个小任务后，必须进行提交。
- 提交消息格式：`feat/fix/chore(<模块>): <描述>`。
- 使用 `git notes` 记录每个任务的详细摘要（如果环境支持）。

## 3. 阶段性验收 (Phase Completion)
每个阶段 (Phase) 完成后，必须执行以下“用户手册验证协议”：
- 确认该阶段产出符合 `spec.md` 中的验收标准。
- 更新项目进度状态。
- 如果涉及 UI，需进行交互验证。

## 4. 质量检查点
- **算法对比验证**：每次优化切分算法后，需与“标准切分案例库”进行对比。
- **Lint 检查**：提交前必须通过 Python (Black) 和 React (ESLint) 的检查。
