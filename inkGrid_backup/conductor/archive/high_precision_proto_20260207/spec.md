# Track Specification: 高精度切分原型与标注界面

## 1. 概述
本赛道旨在构建一个端到端的原型系统，集成 PaddleOCR 用于字符检测和 SAM 用于笔画级分割，并提供一个基于 React 的 Web 界面供用户校准边界。这是实现“极致精准”目标的核心基础。

## 2. 功能需求
### 后端 (Python/FastAPI)
- **OCR 引擎**: 集成 PaddleOCR，实现碑帖全图的字符检测（生成中心点和初步 BBox）。
- **分割引擎**: 集成 SAM (Segment Anything Model)，接收 BBox 或点提示，生成精确的笔画边缘掩码。
- **图像服务**: 提供原始图像切片和处理后预览图的 API。
- **导出模块**: 按照“汉字+拼音+序号”规则保存最终切分结果。

### 前端 (React)
- **交互画布**: 支持加载高清碑帖，可视化显示 OCR 检测到的所有字符框。
- **精准校准**: 允许用户通过鼠标拖拽微调 BBox。
- **实时分割预览**: 用户调整框后，前端请求后端 SAM 引擎实时更新笔画分割效果。

## 3. 非功能需求
- **精度**: 分割边缘需贴合笔画，误差控制在像素级。
- **响应速度**: SAM 推理响应应在 1s 内，确保交互流畅。

## 4. 验收标准
- 能够加载 `峄山碑.jpg` 并成功检测出绝大部分字符。
- 用户能够通过界面修正至少 5 个复杂字符的边界。
- 导出的单字图片完整包含所有笔画，且无相邻文字干扰。
